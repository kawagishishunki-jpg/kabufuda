# 実装計画書：株札カード対戦ゲーム「株ポーカー」Webアプリ開発

「株札こいこい」と「ポーカー」を融合させた斬新なゲーム「株ポーカー」のWebアプリ（プロトタイプ）開発に向けた実装計画を提案します。
本アプリは、PCやスマートフォンからブラウザでアクセスし、ルームコードを共有することで友達同士でリアルタイム対戦ができるほか、CPU対戦やローカル対戦（パス＆プレイ）でルールをすぐに検証できるものとします。

## ユーザー確認・検討事項

実装を進めるにあたり、以下のルール仕様および設計についてご確認・ご意見をいただけますと幸いです。

> [!IMPORTANT]
> ### 1. 「1.2」の特殊ルール（10.10キラー）の挙動について
> 役の強さにおいて「最強は10.10。ただし1.2には負ける」とあります。この挙動について以下の認識で相違ないでしょうか？
> - 通常時（場に10.10がいない場合）：「1.2」は単なる「3点（1+2=3）」として扱われ、他の役や高い点数の手（例: 9点、ゾロ目など）には敗北する。
> - 対決時（場に10.10と1.2が同時に存在する場合）：本来最強である10.10に対して、1.2を所持しているプレイヤーが勝利する。
> - ※もし3人以上の対戦で「10.10」「1.2」「9.9」が同時にショーダウンした場合の勝敗はどうなるでしょうか？（例：1.2は10.10に勝ち、10.10は9.9に勝ち、9.9は1.2に勝つという三すくみ状態になります。この場合の優先順位や処理方法についてご指定があれば教えてください）

> [!IMPORTANT]
> ### 2. ベットラウンドの進行ルールについて
> 「3ターンのベットラウンド（1〜3チップ、チェック不可）」について、以下のアクション進行を想定しています。
> - 各プレイヤーは自分のターンで「フォールド（降りる）」「コール（前の人と同じ額を出す）」「レイズ（前の人より多く上乗せして出す：1〜3チップの範囲）」のいずれかを選択。
> - 「チェック（賭け金ゼロでパス）」は不可。
> - 3ターン（3ラウンド）のベット終了後、残ったプレイヤーでショーダウン（手札公開）。
> - ※手札はゲーム開始時に配られた2枚のままで、ラウンド間でのカードの追加ドローや交換は行わないルール（スタンドパット）でよろしいでしょうか？

> [!IMPORTANT]
> ### 3. お祝儀ルールの詳細について
> - **0点で全員を下ろした時（各プレイヤーから3チップ）**：
>   - ショーダウンせずに全員をフォールドさせた時点で、自分が「0点（合計値の一桁目が0）」であった場合、ポットとは別に生き残っていた（あるいは参加していた）全員から3チップずつ徴収する、という解釈でよろしいでしょうか？
> - **1.1で勝利時のルーレット（山札1枚引き×3チップ）**：
>   - 勝利後、山札からランダムに1枚カードを引き、その数値（1〜10）の3倍のチップを敗者全員（または各敗者）から受け取る、という解釈でよろしいでしょうか？
> - **10.10で勝利時（各プレイヤーから20チップ）**：
>   - 10.10で勝利した際、敗者全員から20チップずつ受け取る、という解釈でよろしいでしょうか？

## 技術スタックの提案

オンラインマルチプレイをサーバー費用をかけずに手軽に実現するため、以下の構成を提案します。

1. **フロントエンド**: React (Vite) + Tailwind CSS または Vanilla CSS
2. **通信方式 (P2P)**: **PeerJS (WebRTC)**
   - サーバーを介さず、プレイヤーのブラウザ同士が直接通信（P2P）します。
   - ホストプレイヤーがルームを作成すると「ルームコード」が生成され、ゲストプレイヤーはそのコードを入力することでホストのゲームに接続します。
   - サーバー維持費が完全に無料となり、静的配信（GitHub PagesやVercelなど）のみで全世界に公開可能です。
3. **対戦モード**:
   - **オンライン対戦モード**: ルームコードを使って友達と対戦。
   - **CPU対戦モード**: 1人で役や流れを確認するためのモード（AIが簡単な確率思考でベット・フォールドを判断）。
   - **ローカル対戦モード**: 1台の端末を交互に操作して遊ぶモード。
4. **デザイン**:
   - 和風モダン（黒・赤・金を基調とした漆器や襖絵を想起させる高級感あるデザイン）。
   - カードは「株札」のクラシックな絵柄をベースにした美しいオリジナルデジタルカードデザイン。
   - ベット時や勝利時の豪華なエフェクト・アニメーション。

## 開発・検証プラン

### 1. 役判定エンジンとCPU対戦の実装
まずはオフラインで動作するゲームコアロジックを作成します。
- 21枚の山札生成、手札配布。
- 強さの比較ロジック（`10.10` と `1.2` の特殊相性を含む）。
- CPUとの対戦モードの実装（UI構築と基本的なゲーム進行制御）。

### 2. UIデザインと演出の強化
- 和風グラフィック（株札の絵柄、筆文字フォント、襖の背景など）の適用。
- チップがポットに集まるアニメーション、お祝儀ルーレットの演出。

### 3. P2Pオンラインマルチプレイの実装
- PeerJSを組み込み、ルームコードによるマッチングを可能にします。
- 同一ブラウザの別タブ（または別ウィンドウ）で2人プレイができるようにし、デバッグを行います。

### 4. 自動テスト・検証
- `evaluator.js`（役判定ロジック）に対する単体テストを作成し、全役の強さ関係や特殊ルール（1.2 vs 10.10）が仕様通り動作することを確認します。
- テストコマンド: `npm run test` (Vitestを使用)
